## 2 何谓模块解析

![2020-02-13-16-26-41](../../assets/43.1.png)

模块解析包括：**抽象语法树分析**、**依赖分析**、**模块语法替换** 等。



## 3 不做模块解析的影响

![2020-02-13-16-28-10](../../assets/43.2.png)

如果某个模块不做解析，该模块经过 `loader` 处理后的代码就是最终代码。

如果没有 `loader` 对该模块进行处理，该模块的源码就是最终打包结果的代码。

如果不对某个模块进行解析，可以缩短构建时间。



## 4 不需要解析的模块

模块中无其他依赖：一些已经打包好的第三方库，比如 `jQuery`。



## 5 不解析某个模块的方法

配置 `module.noParse`，它是一个正则表达式，被正则匹配到的模块将不参与解析。



## 6 实测备忘

:one: 错将 `noParse` 直接写到 `webpack.config.js` 的一级选项下，导致报错。应写到 `module` 选项下：

```js
module.exports = {
  module: {
    noParse: /jquery/
  },
}
```

且写与不写都不会影响 `jQuery` 的正常使用。
